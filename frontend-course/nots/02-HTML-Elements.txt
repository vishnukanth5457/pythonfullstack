## üîπ What is an HTML Element?

An **HTML element** is **everything from the opening tag to the closing tag**, including the content inside.

--> Basic structure

```html
<tagname>Content</tagname>
```

--> Example

```html
<p>This is a paragraph</p>
```

üìå Here:

* `<p>` ‚Üí opening tag
* `</p>` ‚Üí closing tag
* `This is a paragraph` ‚Üí content
* **Together = one HTML element**

----------------------------------------------------------------------------------------------------------------------

## üß† Real-world analogy

Think of an element like a **box** üì¶

* Opening tag = opening the box
* Content = item inside the box
* Closing tag = closing the box

----------------------------------------------------------------------------------------------------------------------

## üîπ Opening & Closing Tags

--> Opening Tag

```html
<h1>
```

--> Closing Tag

```html
</h1>
```

--> Full Example

```html
<h1>Welcome to HTML</h1>
```

üìå Closing tag always has `/`

## üîπ Empty / Self-Closing Elements

Some elements **do not have content**, so **no closing tag** is needed.

--> Common empty elements

* `<br>` ‚Äì line break
* `<hr>` ‚Äì horizontal line
* `<img>` ‚Äì image
* `<input>` ‚Äì form input
* `<meta>` ‚Äì metadata
* `<link>` ‚Äì external resource

--> Example

```html
<p>Hello<br>World</p>
```

--> Image example

```html
<img src="logo.png" alt="Company Logo">
```

üìå These elements are also called:

* **Void elements**
* **Self-closing elements**

‚ùå Wrong:

```html
<img></img>
```

----------------------------------------------------------------------------------------------------------------------

## üîπ Nested Elements

When one element is placed **inside another**, it‚Äôs called **nesting**.

--> Correct nesting

```html
<p>This is <strong>important</strong> text.</p>
```

--> Rule

> Tags must be closed in the **reverse order** they were opened

--> ‚ùå Incorrect nesting

```html
<p>This is <strong>wrong</p></strong>
```

--> ‚úÖ Correct nesting

```html
<p>This is <strong>correct</strong></p>
```
